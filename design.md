## 1. 项目概述

### 1.1 项目背景

**LexiconPrep** 是一款专为考研学子打造的学情管理与备考平台。针对考研备考周期长、压力大、知识点细碎的痛点，通过暗黑终端美学设计营造沉浸式学习氛围，结合科学的数据可视化与间隔重复算法，帮助用户建立秩序感，缓解备考焦虑，提升复习效率。

### 1.2 项目目标

| 目标类型 | 描述 |
|---------|------|
| **用户体验** | 暗黑终端美学 + WebGL 沉浸式背景，减少视觉疲劳，营造专注氛围 |
| **功能目标** | 智能题库、错题本（间隔重复）、闪卡复习、番茄钟专注、社区互动、管理后台 |
| **数据目标** | 通过实时 DB 聚合查询提供学情洞察和学习建议 |
| **性能目标** | 首屏加载时间 < 2s，API 响应 < 500ms |

### 1.3 项目范围

**核心功能模块：**
- 首页（WebGL 矩阵背景 + 逐字解密动画）
- 认证系统（注册/登录/JWT 刷新）
- 个人主页（日历打卡/信息编辑/密码修改）
- 资源库与搜索 + 错题本
- 专注做题模式（即时判分 + 解析）
- 闪卡复习（间隔重复）
- 学习进度追踪（实时统计图表）
- 社区互动（发帖/点赞/排行榜）
- 番茄钟专注（悬浮 + 全屏）
- 管理后台（题目 CRUD/用户管理/系统统计）

**非功能范围：**
- 暂不支持实时视频课程
- 暂不支持在线支付功能
- 暂不支持浅色主题切换

### 3.2 核心模块划分

| 模块 | 职责 | 核心端点 |
|------|------|---------|
| **Auth** | 用户认证 | register, login, refresh |

| **Questions** | 题库与答题 | list, detail, answer |
| **Mistakes** | 错题本 | list, delete, review |
| **Flashcards** | 闪卡系统 | CRUD, review |
| **Resources** | 资源库 | list, detail |
| **Stats** | 学习统计 | overview, progress, trend |
| **Community** | 社区互动 | posts, like, leaderboard |
| **Admin** | 管理后台 | questions CRUD, users, stats |

### 3.3 关键交互流程

#### 答题流程

```
用户选择答案 → POST /questions/{id}/answer
  → 排序后字符串比对答案
  → 写入 study_logs（含 time_spent）
  → 答对：
    → 查询是否首次答对 → 去重更新 total_knowledge_points
    → 查询是否在错题本 → 标记 mastered=True
  → 答错：
    → 查询是否已有错题 → 有则 wrong_count+1, mastered=False
    → 无则新建错题记录（next_review_at = 1天后）
  → 返回 { is_correct, correct_answer, analysis }
```

#### Token 刷新流程

```
前端请求 → 401 响应 → 检查 refresh_token
  → 无 refresh_token → 清除认证 → 跳转登录
  → 有 refresh_token → POST /auth/refresh
    → 成功 → 更新 localStorage → 重试原请求
    → 失败 → 清除认证 → 跳转登录
  → 并发请求 → 加入等待队列 → 刷新完成后统一重试
```

### 5.1 RESTful API 规范

| 规范项 | 规则 |
|--------|------|
| 基础路径 | `/api/v1` |
| 版本控制 | URI 中包含版本号 |

| 响应格式 | JSON |
| 错误格式 | `{"detail": "错误描述"}` |
| 分页 | `limit` + `offset` 参数 |

## 7. 安全策略

### 7.1 已实现的安全措施

| 措施 | 实现方式 | 说明 |
|------|---------|------|
| JWT 认证 | python-jose | access 15min + refresh 7day，type 字段防混用 |
| 密码加密 | passlib bcrypt | CryptContext 自动加盐 |
| 登录限流 | slowapi | 5 次/分钟，防暴力破解 |
| 全局限流 | slowapi | 60 次/分钟（可配置） |
| CORS | 动态配置 | 从 ALLOWED_ORIGINS 环境变量读取 |
| 搜索注入防护 | escape_search | 转义 `%` 和 `_` 通配符 |
| 点赞防刷 | PostLike 联合唯一 | 同一用户同一帖子只能点赞一次 |
| 头像校验 | 大小 + MIME | ≤500KB，必须 `data:image/` 开头 |
| 密码复杂度 | Pydantic validator | ≥8位 + 大写 + 小写 + 数字 |
| 帖子长度 | max_length=2000 | 防超长内容 |
| 昵称唯一性 | 注册时校验 | 防止昵称冲突 |
| 管理员自保护 | 角色修改检查 | 不允许修改自己的角色 |
| 级联删除 | 删除题目时清理 | 同时删除关联 mistakes + study_logs |
| 环境变量 | .env + pydantic-settings | 敏感配置不硬编码 |

### 7.2 待实现的安全措施

| 措施 | 优先级 | 说明 |
|------|--------|------|
| HttpOnly Cookie | P0 | refresh_token 应存于 HttpOnly + Secure + SameSite Cookie |
| JWT 密钥校验 | P0 | 启动时检查 JWT_SECRET_KEY 非默认值 |
| HTTPS | P0 | 生产环境必须启用 TLS |
| XSS 防护 | P1 | 社区帖子输出时 HTML 转义，前端使用 v-text |
| CSRF 防护 | P1 | Cookie 认证后需添加 CSRF Token |
| 邮箱验证 | P2 | 注册后发送验证邮件 |
| 密码重置 | P2 | 实现忘记密码流程 |

## 8. 部署方案

### 8.1 开发环境

```bash
# 前端
npm install
npx vite --port 3000

# 后端（SQLite 零依赖模式）
cd backend
pip install -r requirements.txt
python init_db.py && python seed.py
uvicorn app.main:app --port 8000 --reload
```

### 8.2 生产环境

```bash
# 1. 启动基础设施
docker compose up -d

# 2. 后端
cd backend
cp .env.example .env
# 编辑 .env：设置 DATABASE_URL, JWT_SECRET_KEY, ALLOWED_ORIGINS 等
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4

# 3. 前端
npm run build
# 使用 nginx 托管 dist/ 目录
```

## 10. 风险评估与应对策略

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| JWT 密钥泄露 | 高 | 严重 | 启动时校验非默认值；生产环境使用随机密钥；定期轮换 |
| XSS 攻击 | 中 | 严重 | 社区帖子 HTML 转义；前端使用 v-text；CSP 策略 |
| 暴力破解 | 中 | 中等 | 已实现登录限流 5 次/分钟；可增加验证码 |
| 数据库连接耗尽 | 低 | 中等 | 已配置连接池（pool_size=10, max_overflow=20, pool_pre_ping） |
| 构建体积过大 | 高 | 中等 | 手动 chunk 分割；Element Plus/Chart.js 按需引入 |
| 种子数据不足 | 高 | 低 | 扩充至 500+ 题；接入真实资源文件 |
| 版权风险 | 中 | 严重 | 确认题目无版权问题；使用原创或授权内容 |
| 移动端兼容性 | 中 | 中等 | 优化响应式布局；低端设备降级 WebGL 特效 |
